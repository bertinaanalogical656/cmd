## 4. Что умеет

### Один CLI на флот серверов

Подключил CLI к одному серверу — он работает на всех остальных через встроенный proxy. Одна подписка вместо трёх. Никто из конкурентов так не умеет — потому что это противоречит их монетизации.

### Async — запустил и забыл

Поставил задачу с телефона, закрыл приложение, поехал дальше. Агент работает на твоём сервере столько, сколько надо — час, два, всю ночь. Вернулся — посмотрел результат.

В отличие от Telegram-ботов, где задача умирает через 5 минут с фейковым «✅ выполнено».

### Голосовой ввод

Печатать длинные промпты на телефоне — мучение. Нажал кнопку, продиктовал, отправил. Web Speech API — Google Speech распознаёт прямо в браузере, голос на наши сервера не уходит.

Работает в Safari iOS 14.5+, Chrome, Edge, Samsung Internet. Firefox пока нет — но это ~3% юзеров.

### Multi-project, multi-vendor

Любое количество проектов на любом количестве серверов с любым агентом.

Сегодня поддерживаем **Claude Code** и **Gemini CLI**. На очереди: Codex, Aider, Cursor CLI и любой другой по запросу комьюнити (PR welcome).

### Self-hosted: ключи у тебя

OAuth Anthropic, GEMINI_API_KEY, контекст агентов, история сессий — всё хранится только в твоей инфраструктуре. Никаких third-party sync-серверов, никаких прокси через нас. Open-source — `git clone` и проверь сам.

## 6. Безопасность — всё остаётся у тебя

Если ты ставишь self-host, у нас нет никаких твоих данных. От слова совсем.

- **API-ключи и OAuth-токены** хранятся только в твоей Postgres-базе. Не уходят к нам, не уходят к Anthropic-side broker, не уходят никуда кроме вызова к самому Claude/Gemini API.
- **Контекст агентов и история сессий** — тоже только в твоей БД.
- **Агент на сервере физически не может прочитать `~/.claude/*`** — это заблокировано на уровне протокола, даже если ты или агент попросят.
- **Нет открытых портов.** Агент инициирует исходящее WebSocket-соединение к мастер-узлу. Никаких exposed-портов, никаких dangerous-publish, никакой Cloudflare-tunnel магии.
- **Open source, AGPL-3.0.** Не верь на слово — `git clone` и проверь как именно мы обращаемся с твоими ключами.

В облачной версии — то же самое плюс encryption at rest: ключи зашифрованы в БД, расшифровываются только в момент запуска агента. Если ты не доверяешь нам — поднимай self-host.

## 8. FAQ

### Это безопасно? Куда уходят мои API-ключи?

Никуда. В self-host-варианте — они хранятся только в твоей Postgres-базе. В облачной — в нашей зашифрованной БД, расшифровывается только в момент запуска агента. Open source — можешь проверить.

### Работает без VPN?

Да. Если ты ставишь self-host на свой VPS — UI ходит к твоему серверу напрямую. Облачная версия тоже доступна без VPN.

Один нюанс: Gemini CLI Google блокирует для российских IP. Если твой сервер не в РФ — Gemini работает. На устройстве (телефон) ничего не нужно — связь идёт сервер ↔ Anthropic/Google.

### Какие именно CLI поддерживаются?

Сегодня в продакшене: **Claude Code** и **Gemini CLI**.

В разработке: Codex CLI (OpenAI), Aider, Cursor CLI.

Хочешь добавить свой — открой issue или PR. Архитектура rасширяемая.

### Чем отличается от Anthropic Remote Control?

Anthropic привязан только к Claude и к подписке Anthropic. У тебя — любой CLI, любой сервер, без привязки к вендору. Плюс мы self-hosted — Anthropic Remote Control работает только через их инфраструктуру.

Если тебе хватает только Claude и одного устройства — Anthropic Remote OK. Если у тебя несколько серверов, или нужны другие CLI — мы.

### Открытые порты на серверах?

Не нужны. Агент инициирует исходящее WebSocket-соединение к мастеру (или к нашему облаку). Входящие соединения не открываются — никаких exposed-портов, никаких dangerous publish.

### А что если Anthropic выпустит свой Claude Code Web?

Они уже выпустили. Только он работает на их сэндбоксах, не на твоих серверах, и только с Claude. Это другая категория — у них «cloud sandbox для casual use», у нас «self-hosted control plane для твоего флота».

### Как с командной работой?

Сейчас — один пользователь, одна инфраструктура. Команды, SSO и аудит — на roadmap, не сейчас.
